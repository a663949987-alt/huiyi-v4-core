# 会意 v4.1.2 局域网更新报告

## 已实现

- 证据包与 parser 报告导出到手机公共下载目录：`Download/HuiyiV4/`。
- 设置页新增局域网更新区域：
  - 更新地址输入，允许留空
  - 检查更新，留空时自动发现同一 Wi-Fi 下的 `8787/latest.json`
  - 下载 APK
  - 打开系统安装确认
- 支持读取 `latest.json`。
- 支持相对 APK 地址，例如 `apkUrl: huiyi-v4.1.2-debug.apk`。
- 下载后校验 SHA-256。
- 不做静默安装，仍由用户确认。
- 新增电脑侧局域网静态服务脚本：`scripts/start-lan-update-server.ps1`。
- 新增一键发布脚本：`scripts/publish-lan-update.ps1`，每次下发更新包时自动构建、复制 APK、计算 SHA-256、刷新 `latest.json`。

## 使用方式

以后每次下发新包，开发侧先跑：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/publish-lan-update.ps1
```

它会自动刷新：

- `outputs/update_server/latest.json`
- `outputs/update_server/huiyi-v<versionName>-debug.apk`

然后启动局域网更新服务：

1. 电脑运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-lan-update-server.ps1
```

2. 手机和电脑连接同一 Wi-Fi。

3. 手机 App 设置页的更新地址可以留空。

4. 点击“检查”，App 会自动发现局域网更新服务。

5. 自动发现成功后，点击“下载” -> “打开安装”。

如果自动发现失败，再手动填写：

```text
http://电脑局域网IP:8787/latest.json
```

## 输出

- update manifest：`outputs/update_server/latest.json`
- LAN APK：`outputs/update_server/huiyi-v4.1.2-debug.apk`
- debug APK：`outputs/huiyi-v4.1.2-debug.apk`
- release APK：`outputs/huiyi-v4.1.2-release-preconfigured.apk`

## 签名说明

局域网更新当前使用 debug APK，适合你的一台手机开发测试。release 仍未正式签名。后续如果要长期自用分发，需要配置正式 keystore，并保持签名一致，否则 Android 会拒绝覆盖安装。
